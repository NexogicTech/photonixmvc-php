# 访问已定义的路由

入口与重写：

- 入口文件为 `public/index.php`，其中执行 `Routes::runRouter()`。
- Apache 与 Nginx 均通过重写将未命中静态文件的请求转发到入口。
- 可选 `public/router.php` 用于统一静态文件处理与回退到入口。

URL 与匹配：

- 精确匹配：`/`、`/about`、`/docs/router`。
- 参数化匹配：`/hello/{name}`、`/post/{slug}`，占位符匹配单段 `[^/]+`。
- 中文参数支持：路径在调度前进行 URL 解码，可直接访问 `/你好/hello/` 等形式。

返回：

- 命中后调用控制器方法，返回 `HtmlResponse` 内容或字符串将被输出。
- 未命中返回 `config/error.php` 中定义的 404 页面；异常或缺类返回 500。

示例：

```
GET /              -> App\\Home\\Controller\\Index::index
GET /hello/Photonix -> App\\Home\\Controller\\Index::hello('Photonix')
GET /你好/hello/     -> App\\Home\\Controller\\Index::hello('你好')
```
